Trade vs OrderBook

Dos tipos de datos de mercado fundamentalmente diferentes

Trade (Transaccion)

Un trade es una operacion ya ejecutada. Alguien compro y alguien vendio a un precio concreto. Es un hecho que ya ocurrio.

timestamp: 1711800000
symbol: "BTCUSDT"
price: 87,432.50
volume: 0.15 BTC
side: BUY
  • ✓ Representa una operacion real y cerrada
  • ✓ Tiene precio, volumen y direccion (BUY/SELL)
  • ✓ Sirve para medir presion compradora/vendedora
  • ✓ Analogia: el ticket de caja de una compra

OrderBook (Libro de Ordenes)

El orderbook es la lista de ordenes pendientes. Muestra a que precios la gente quiere comprar (bids) y vender (asks). Es intencion, no hechos.

BIDS (Compra)
87,430 x 2.5
87,425 x 1.8
87,420 x 4.1
↓ mas abajo...
ASKS (Venta)
87,435 x 1.2
87,440 x 3.0
87,450 x 0.7
↑ mas arriba...
  • ✓ Representa intenciones de compra y venta
  • ✓ Bids = a cuanto quieren comprar / Asks = a cuanto quieren vender
  • ✓ Sirve para detectar absorcion y liquidez
  • ✓ Analogia: la cola de gente esperando para comprar/vender

Trade = lo que ya paso   •   OrderBook = lo que la gente quiere que pase

Juntos permiten al modelo detectar patrones como absorcion (muchas ventas en el book pero trades compradores las absorben) o barridos de liquidez (trades que agotan niveles enteros del orderbook).

Codigo Implementado

Fragmentos clave del scraper en funcionamiento

main.py — Entry Point 
import asyncio

async def main():
    brokers: list[BrokerClient] = [
        BybitBroker.create(),
        BinanceBroker.create(),
        KrakenBroker.create(),
        BybitOrderBook.create(),
        BinanceOrderBook.create(),
        KrakenOrderBook.create(),
    ]

    # Conectar todos los brokers concurrentemente
    await asyncio.gather(
        *[b.connect() for b in brokers]
    )

    try:
        while True:
            await asyncio.gather(
                *[b.onListen() for b in brokers]
            )
    except KeyboardInterrupt:
        await asyncio.gather(
            *[b.close() for b in brokers]
        )

class Side(enum.Enum):
    BUY = "BUY"
    SELL = "SELL"

class TSTrade:
    timestamp: int
    name:      str
    price:     float
    volume:    float
    side:      str
    broker:    str

    def __str__(self) -> str:
        return (
            f"TSTrade(ts={self.timestamp}, "
            f"'{self.name}', {self.side} "
            f"price={self.price}, "
            f"vol={self.volume:.8f}, "
            f"broker='{self.broker}')"
        )

class OrderLevel:
    price:    float

Deteccion de Patrones (ML)

Patrones que el modelo de Deep Learning identifica para HFT

📈 Absorcion

Si hay muchas ordenes de venta y el precio no baja, la red detecta que hay un comprador gigante absorbiendo esas ordenes. El precio va a rebotar.

⚡ Barrido de Liquidez

Muchas compras que agotan las ordenes del book. La red entiende que hay que comprar rapido para seguir el impulso del movimiento.

🔴 Presion de Ventas

Volumen de ventas 3x mayor al de compras en los ultimos 10 segundos. La red entiende que el precio va a caer.

Clase 0

Precio Sube

El precio subira mas de un X% en los proximos Y segundos.

Clase 1

Precio Baja

El precio bajara mas de un X% en los proximos Y segundos.

Clase 2

HOLD / Neutral

El precio se mantendra estable. No hay senal clara de movimiento.
